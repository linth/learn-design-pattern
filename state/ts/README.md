


# 狀態圖 (state diagrams) vs. 狀態設計模式 (state design pattern)
狀態圖（State diagrams）和狀態設計模式（State Design Pattern）是相關的概念，但並不完全相同。

## 狀態圖
狀態圖，也被稱為狀態機（State machines），是對系統的狀態和轉換進行視覺化表示的工具。它們用於基於內部狀態和外部事件來建模對象或系統的行為。狀態圖由狀態、轉換和與每個狀態和轉換相關聯的動作組成。

## 狀態設計模式
狀態設計模式（State Design Pattern）是一種軟體設計模式，有助於根據對象的內部狀態來管理其行為。它提供了一種將特定於狀態的行為封裝到獨立類別中的方式，並允許對象在運行時通過改變內部狀態來改變其行為。

在狀態設計模式中，對象的不同狀態由獨立的狀態類別表示，並且對象將其行為委託給當前狀態對象。狀態類別實現了一個共同的介面，使得對象可以在不同狀態之間動態切換。

儘管狀態圖提供了系統中狀態和轉換的視覺表示，但狀態設計模式提供了一種靈活且易於維護的方式來實現與這些狀態相關聯的行為。

總結起來，狀態圖是一種用於描述系統行為的建模工具，而狀態設計模式是一種軟體設計模式，有助於根據對象的內部狀態來管理其行為。狀態設計模式可以用來實現狀態圖中描述的行為，但這不是唯一的方法。
